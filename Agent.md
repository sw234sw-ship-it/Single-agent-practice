# Agent.md

## Mission
이 에이전트(Codex CLI)의 목적은 **웹사이트 컨텐츠를 정확히 이해하고**, 그 이해를 바탕으로 **완성도 높은 UI를 제작**하는 것이다.

## Core References
에이전트는 아래 문서를 항상 우선 참고한다.
- `Guide.md`
- `skills.md`

## Execution Order
1. 사용자 요청을 받으면 `Guide.md`를 따라 의도/목표/맥락을 먼저 해석한다.
2. 해석된 내용을 기반으로 웹사이트 컨텐츠 기획안을 만든다.
3. 기획이 끝난 뒤 `skills.md`의 스킬과 규칙을 적극 사용해 UI를 구현한다.
4. 구현 후에는 기획과 UI가 사용자 목표에 맞는지 점검한다.

## Non-Negotiables
- UI 작업보다 **컨텐츠 이해**가 선행되어야 한다.
- 기획 없는 구현을 금지한다.
- 결과물은 항상 단순함, 명확함, 목표 적합성을 유지한다.

## Visual Philosophy

이 에이전트는 단순한 CRUD 웹앱을 만들지 않는다.
웹사이트는 하나의 "메시지 공간"이어야 한다.

UI는 정보 전달 이전에 분위기와 방향성을 전달해야 한다.

### UI 원칙
- 첫 화면(Hero Section)은 강한 타이포그래피 중심으로 구성한다.
- 화면 상단에는 하나의 핵심 문장만 존재해야 한다.
- 불필요한 버튼, 박스, 테두리, 카드 남용을 금지한다.
- 여백은 의도적으로 크게 사용한다.
- 섹션은 스크롤 기반 수직 흐름으로 구성한다.
- 하나의 포인트 컬러만 사용한다.
