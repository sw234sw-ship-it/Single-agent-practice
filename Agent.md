# Agent.md

## Work Activation Protocol (필수 실행 절차)

이 에이전트는 사용자의 요청을 받으면 즉시 구현을 시작하지 않는다.

### 단계 1. 계획 수립 (Planning Phase)
- 사용자 요청을 받으면 먼저 구현 없이 "계획"을 작성한다.
- 계획에는 최소 아래 내용이 포함되어야 한다:
  - 목표 요약
  - 범위 (In / Out)
  - 화면 구조 또는 기능 구조
  - 예상 작업 단계
  - 완료 기준

- 계획 작성 후 반드시 사용자 승인을 요청한다.
- 승인(OK, 승인, 진행 등 명시적 동의) 전에는 절대 구현을 시작하지 않는다.

### 단계 2. 승인 후 문서 생성
사용자 승인 이후에만 아래 3개의 문서를 생성하고 저장한다.

1) Plan.md
   - 승인된 계획 내용을 정리
2) ContextLog.md
   - 왜 이런 결정을 했는지, 참고한 근거, 제약 사항 기록
3) Checklist.md
   - 작업 단계별 완료 추적

이 3문서는 작업의 단일 기준(Source of Truth)이 된다.

### 단계 3. 작업 시작
- 구현을 시작하기 전에 방금 생성된 3문서를 다시 읽고 현재 목표를 요약한다.
- 그 이후에만 실제 구현 작업을 시작한다.

### 변경 요청이 들어올 경우
- 즉시 구현을 수정하지 않는다.
- 먼저 기존 Plan을 수정한 뒤 사용자 승인 요청
- 승인 후 ContextLog와 Checklist를 업데이트
- 그 다음에 구현 반영
- 승인되지 않은 계획에 기반한 구현은 무효로 간주한다.
