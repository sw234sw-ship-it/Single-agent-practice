# AGENTS.md

## HARD EXECUTION GATE

이 파일은 프로젝트의 최상위 실행 규칙이다.
이 문서의 규칙은 모든 다른 문서보다 우선한다.

이 규칙을 따르지 않은 구현은 무효로 간주한다.

# Agent.md

## Work Activation Protocol (필수 실행 절차)

이 에이전트는 사용자의 요청을 받으면 즉시 구현을 시작하지 않는다.

### 단계 1. 계획 수립 (Planning Phase)
- 사용자 요청을 받으면 먼저 구현 없이 "계획"을 작성한다.
- 승인 전에는 파일 생성, 파일 수정, 코드 작성, 커밋을 금지한다.
- 계획에는 최소 아래 내용이 포함되어야 한다:
  - 목표 요약
  - 범위 (In / Out)
  - 화면 구조 또는 기능 구조
  - 예상 작업 단계
  - 완료 기준

- 계획 작성 후 반드시 사용자 승인을 요청한다.
- 승인(OK, 승인, 진행 등 명시적 동의) 전에는 절대 구현을 시작하지 않는다.

### 단계 2. 승인 후 문서 생성
사용자 승인 이후에만 아래 3개의 문서를 생성하고 저장한다.

1) Plan.md
   - 승인된 계획 내용을 정리
2) ContextLog.md
   - 왜 이런 결정을 했는지, 참고한 근거, 제약 사항 기록
3) Checklist.md
   - 작업 단계별 완료 추적

이 3문서는 작업의 단일 기준(Source of Truth)이 된다.

### 단계 3. 작업 시작
- 구현을 시작하기 전에 방금 생성된 3문서를 다시 읽고 현재 목표를 요약한다.
- 그 이후에만 실제 구현 작업을 시작한다.

### 변경 요청이 들어올 경우
- 즉시 구현을 수정하지 않는다.
- 먼저 기존 Plan을 수정한 뒤 사용자 승인 요청
- 승인 후 ContextLog와 Checklist를 업데이트
- 그 다음에 구현 반영
- 승인되지 않은 계획에 기반한 구현은 무효로 간주한다.

### Checklist Structure (고정 구조)

Checklist.md는 반드시 아래 구조를 따른다.

## Now (현재 실행 중 1개만)
- [ ] 하나의 항목만 존재해야 한다.

## Next
- [ ] 다음 실행 예정 항목들

## Done
- [x] 완료된 항목들

## Backlog
- [ ] 아직 시작하지 않은 후보 작업들

### 구조 규칙

- Now에는 항상 1개만 존재한다.
- Now가 비어 있으면 작업을 시작할 수 없다.
- Now가 완료되면:
  1) Done으로 이동
  2) Next의 첫 항목을 Now로 이동
  3) ChangeLog 기록
- 여러 작업을 동시에 Now에 두는 것을 금지한다.
- Backlog는 우선순위에 따라 Next로 이동 가능하다.

## Checklist Execution Engine (강제 실행 규칙)

Checklist.md는 단순 기록이 아니라 실행 순서를 통제하는 기준 문서다.

### 실행 규칙

1. Checklist에 있는 항목은 반드시 "위에서 아래 순서대로" 하나씩 수행한다.
2. 동시에 여러 항목을 처리하지 않는다.
3. 하나의 항목이 완료되면 즉시:
   - 해당 항목을 [x]로 변경
   - 완료 시간 기록
   - 다음 실행할 항목을 "Now" 섹션으로 이동
4. 새로운 작업이 추가되면:
   - Checklist.md에 즉시 반영
   - 우선순위에 따라 위치 재조정
5. 작업이 삭제되거나 변경되면:
   - Checklist에서 제거 또는 수정
   - ContextLog에 변경 사유 기록

### 강제 규칙

- Checklist 업데이트 없이 다음 작업으로 넘어가는 것을 금지한다.
- 모든 작업 단위 실행 직후 Checklist를 업데이트해야 한다.
- Checklist는 항상 최신 상태여야 한다.

## Change Logging Protocol (자동 수정 기록)

모든 작업 세션에서 코드/파일 수정이 발생하면,
즉시 ChangeLog.md에 기록한다.

### 기록 형식

### YYYY-MM-DD HH:MM - 작업 요약
- 수정 파일:
- 변경 내용:
- 변경 이유:
- 관련 Plan 항목:
- 관련 Checklist 항목:

### 규칙

- 변경 기록 없이 작업을 종료하는 것을 금지한다.
- 하나의 Checklist 항목이 완료될 때마다 ChangeLog를 기록한다.
